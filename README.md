# Betwixt and Between

A visual worldbuilding OS for fiction writers. Manage characters, locations, events, acts, scenes, and notes in a floating-window desktop interface — where every entity is hyperlinked to every other.

Click a character in the Story Graph and their wiki page opens. Click a location in the Timeline and the World Map zooms to it. There are no dead ends.

**Who it's for:** Writers whose mental model is relational, not linear — fantasy and sci-fi worldbuilders, tabletop RPG designers, anyone who finds Scrivener's outlines hostile to how they actually think.

## Features

| Feature | What it does |
|---|---|
| **Story Graph** | Visual relationship map — characters, locations, events as nodes with typed edges (`allied_with`, `rivals`, `appears_in`, `caused_by`, etc.). |
| **Wiki** | Markdown notes for any entity. Full preview, inline search, linked entity chips. |
| **Timeline** | Acts, scenes, and events on a scrollable timeline. Expand rows to see nested detail. |
| **World Map** | Location cards with linked character and event chips. |
| **Window manager** | Every entity opens in its own floating window. Multiple windows of the same type stay open simultaneously. Taskbar groups them. |

## Running locally

**Prerequisites:** Node.js 24+, npm, and a Postgres connection — either a Neon dev branch (recommended) or local Postgres via Docker (`postgres:17-alpine`).

```sh
npm install
cp .env.example .env       # set DATABASE_URL (see below)
npm run db:migrate
npm run dev
```

Open [http://localhost:5173](http://localhost:5173).

### `DATABASE_URL`

```
# Neon dev branch (recommended)
DATABASE_URL=postgres://user:pass@ep-xxxxx.region.aws.neon.tech/dbname
```

For Cloudflare local runtime emulation, copy `.dev.vars.example` to `.dev.vars` and set the same `DATABASE_URL`. Cloudflare Worker secrets are runtime bindings, so server code reads them from `event.platform.env` during requests; plain `npm run dev` reads from `.env` through SvelteKit's private dynamic env fallback. See [docs/architecture.md → Environment variables](docs/architecture.md#environment-variables).

### Database commands

```sh
npm run db:push       # apply schema changes directly (dev shortcut)
npm run db:generate   # generate a new migration from schema changes
npm run db:migrate    # apply all pending migrations
npm run db:studio     # open Drizzle Studio to browse the database
```

The schema is generated by Drizzle from `src/lib/server/db/schema.ts`. `bump_updated_at()` BEFORE UPDATE trigger keeps `updated_at` fresh — application code does not set this column. See [CLAUDE.md → Conventions](CLAUDE.md#conventions).

### Tests + build

```sh
npm test              # Vitest unit + integration (in-process PGlite, ~5s)
npm run test:e2e      # Playwright against production preview (workers: 1, ~60s incl. build)
npm run check         # svelte-check type pass
npm run build         # production build
npm run preview       # serve build on :4173
npm run cf:preview    # build and serve through Wrangler
```

Playwright browsers need a one-time install: `npx playwright install firefox`.
