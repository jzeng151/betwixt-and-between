## Surface 1 — Settings App (Phase 0a)

### Layout (ASCII)

```
+--[ ● ● ] Settings ─────────────────────────────────────────────+
|                                                                 |
|  +───────────────────+  +────────────────────────────────────+  |
|  |  APPEARANCE       |  | APPEARANCE                          | |
|  |  Hotkeys          |  |                                     | |
|  |  Entity Defaults  |  | THEME                               | |
|  |  Editor           |  |  ( ) Light  (•) Dark  ( ) Auto      | |
|  |  Data             |  |                                     | |
|  |                   |  | ACCENT                              | |
|  |                   |  |  ● ● ● ● ●●● ● ● ● ●  (swatches)    | |
|  |                   |  |  Selected: Amber #c8942a            | |
|  |                   |  |                                     | |
|  |                   |  | DENSITY                             | |
|  |                   |  |  ( ) Compact (•) Normal ( ) Comfy   | |
|  |                   |  |                                     | |
|  +───────────────────+  +────────────────────────────────────+  |
|  720 × 540  (sidebar 200px, content 520px)                      |
+─────────────────────────────────────────────────────────────────+
```

### Pass 1 — Info Architecture
First the user sees the **section list** (left sidebar) so they understand what's configurable. Active section bolded + amber left-border (4px). Then the section's heading (9px Inter 600 uppercase). Then the controls top-down: theme → accent → density (most-used first). No "Save" button — preferences write-through on change (per existing optimistic-save pattern in DESIGN.md).

### Pass 2 — Interaction States
| State | Theme | Accent | Density |
|---|---|---|---|
| Loading | (instant from localStorage; no loading state) | (instant) | (instant) |
| Empty | (n/a — defaults always present) | (n/a) | (n/a) |
| Error (write fail) | Inline 11px `--color-rel-rival`, "Could not save preference, retrying..." | same | same |
| Success | Visual: control snaps to new state immediately | same | same |
| Pending sync (post-T8b) | 12px spinner next to changed control until server confirms | same | same |

### Pass 3 — User Journey
Cold-open user clicks Settings dock icon → window opens with Appearance section pre-selected. They want to change theme: visceral 5sec — instant theme application as they click; behavioral 5min — exploring sections to discover what's tunable; reflective 5yr — Settings becomes a habit (Cmd-, hotkey to open).

### Pass 4 — AI Slop Risk
**Risks:**
- Don't put icons in colored circles next to section names. Section list is text + active-state indicator only.
- Don't use rounded large radii on every control. Match existing window radius (8px) and form-control radii from existing CharacterEditor.
- Theme toggle is a segmented control (3 buttons sharing a border), NOT 3 separate buttons with shadows.

### Pass 5 — Design System Alignment
| Element | Token / pattern |
|---|---|
| Window chrome | Same as all apps: 8px radius, `--color-surface` bg, `--color-surface-2` titlebar, traffic-light buttons |
| Section labels | 9px Inter 600 uppercase 0.12em tracked, `--color-text-muted` |
| Form labels | 12px Inter 500 |
| Body | 12px Inter 400 |
| Accent swatches | 24×24px circles, 4px gap, 1px ring on selected via `outline: 2px solid var(--color-accent); outline-offset: 1px` |
| Active sidebar item | Left 4px border `--color-accent` + text `--color-text` (vs muted) |
| Form sections | 32px vertical gap; 1px `--color-border` separators |

### Pass 6 — Responsive + A11y
- **Min width:** ≥ 1280px (per DESIGN.md global rule). Window itself is 720×540, never below 600 wide.
- **Keyboard:** Tab cycles sidebar items; Arrow Up/Down within sidebar; Enter activates a section. Within content: standard form-tab order. Theme/density radios respond to Arrow Left/Right. All controls have visible focus ring (`outline: 2px solid var(--color-accent)`).
- **Screen reader:** Sidebar = `<nav role="navigation">` with `aria-label="Settings sections"`. Active item `aria-current="page"`. Each control is properly labeled via `<label for>`.
- **Touch targets:** ≥ 44×44px hit area on swatches even though visual is 24px (use padding).

### Pass 7 — Unresolved
- **Where in the dock?** Existing Taskbar.svelte hosts dock icons. Settings goes at the right end (system-tray convention) or alongside other apps? **Lean: right end**, matching macOS/iOS convention.
- **Hotkey?** Cmd-, is the macOS convention; Ctrl-, on Windows/Linux. **Lean: register both**, normalized via hotkey-registry (Phase 1.5).

## Surface 2 — Notes App (Phase 0/1)

### Layout (ASCII)

```
+--[ ● ● ] Notes ────────────────────────────────────────────────+
|                                                                 |
|  +─────────────────+ +─────────────+ +──────────────────────+   |
|  | + New Folder    | | Today      | |                       |   |
|  | + New Note      | | ──────────  | | ## Worldbuilding     |   |
|  |                 | | Plot Hooks  | |                       |   |
|  | ▼ Worldbuilding | | Chapter 3   | | Some thoughts on     |   |
|  |   ▼ Locations   | | ──────────  | | the magic system...  |   |
|  |     - Mountains | | This Week   | |                       |   |
|  |     - Coastal   | | Magic Notes | |                       |   |
|  |   ▼ Politics    | | Naming List | |                       |   |
|  | ▼ Plot Threads  | | ──────────  | | (markdown editor)    |   |
|  |   - Quest A     | | Older       | |                       |   |
|  |   - Quest B     | | Old idea 1  | |                       |   |
|  | ▼ Inbox         | |             | |                       |   |
|  |                 | |             | | [Edit] [Preview]      |   |
|  +─────────────────+ +─────────────+ +──────────────────────+   |
|  900 × 600  (folders 220 / list 240 / editor flex)              |
+─────────────────────────────────────────────────────────────────+
```

### Pass 1 — Info Architecture
First: **folder tree** (left, 220px) — orienting "where am I." Second: **entry list** (middle, 240px) — orienting "what's in here," grouped by recency (Today / This Week / Older) like Apple Notes. Third: **editor** (right, flex) — the doing. Open-folder defaults: tree expanded for the most-recently-edited folder's path, others collapsed. Inbox folder always visible at top-level (catches new notes when no folder is selected).

### Pass 2 — Interaction States
| State | Folder tree | Entry list | Editor |
|---|---|---|---|
| Loading | Skeleton shimmer 3 rows | Skeleton 5 rows | Spinner centered |
| Empty (no folders) | "No folders yet. Click + New Folder." with dimmed [+] button | n/a | n/a |
| Empty (folder w/ no entries) | Folder shows `(0)` badge | "This folder is empty. Click + New Note." | n/a until note picked |
| Empty (no entry selected) | n/a | All entries grouped | "Pick a note from the list to start writing." |
| Error (save fail) | Inline 11px `--color-rel-rival` toast at editor bottom | Same | Same |
| Success (typing) | Auto-save indicator: small dot fades from amber → muted (1s pulse) at top-right of editor |
| First-run | One-time hint: "Brain dump anything — it's just for you." in muted text |

### Pass 3 — User Journey
Visceral 5sec — open Notes, see the familiar three-column Apple-Notes layout, instantly know how to navigate. Behavioral 5min — create folder, drag entry between folders. Reflective 5yr — Inbox becomes the universal capture spot; folders accumulate organically.

### Pass 4 — AI Slop Risk
**Risks:**
- Don't add an "icon-in-colored-circle" next to each note in the list. Notes are text-led, not card-led.
- Don't render entries as cards with shadows. They're text rows in a list.
- Don't auto-pin "Welcome to Notes!" examples on first launch. Empty state copy is enough.
- Don't add a 3-column feature grid anywhere.

### Pass 5 — Design System Alignment
| Element | Token / pattern |
|---|---|
| Folder tree row | 11px Inter 400, 28px row height, 8px indent per nesting level. Active folder bg `--color-surface-2`, text `--color-text` |
| Folder caret (▶/▼) | 10px Inter, `--color-text-muted` |
| Entry row title | 12px Inter 500 |
| Entry row preview | 11px Inter 400 `--color-text-muted`, 1 line ellipsis |
| Time-group label ("Today") | 9px Inter 600 uppercase 0.12em tracked, sticky to top of list |
| Editor | Inter 14px for body (slightly larger for prose), Fraunces 18px for `# Headings` (matches Wiki rendering) |
| Auto-save indicator | 6px circle, `--color-accent` → fade to `--color-text-muted` at 1s |
| Markdown editor | Reuse the existing `marked` setup from Wiki. Inline preview toggle (Edit/Preview tabs) at editor bottom |
| Section separators | 1px `--color-border` between time groups |

### Pass 6 — Responsive + A11y
- Min width 1280px (DESIGN.md global). Window 900×600 default; min 720×400.
- Keyboard: Arrow Up/Down navigates folder tree (Right expands, Left collapses). Tab moves to entry list, then editor. Cmd/Ctrl+N creates note in active folder. Cmd/Ctrl+Shift+N creates folder. Backspace on selected folder/entry prompts delete.
- Screen reader: tree = `<ul role="tree">` with `aria-expanded`; entries = `<ul role="list">` with `aria-current="true"` on selected; editor labeled.
- Touch targets ≥ 44px hit area on tree carets even though visual is 10px.

### Pass 7 — Unresolved
- **Drag entries between folders** — supported in v1 or deferred? Lean: supported, since Apple Notes parallel sets the expectation.
- **Folder rename UX** — double-click vs context-menu? Lean: double-click + Enter to commit (matches Finder).
- **Search bar?** Spec'd in Phase 1.5 Cmd-K. Notes app itself does NOT add an inline search bar; users open Cmd-K instead. Worth confirming.

## Surface 3 — Wiki Rework (Phase 1)

### Layout (ASCII) — Notes-as-Sections model

```
+--[ ● ● ] Wiki ─────────────────────────────────────────────────+
|                                                                 |
|  +─────────────────────+ +────────────────────────────────────+ |
|  | A · ABRAHAM         | | Aragorn   (Fraunces 24px)          | |
|  | A · ANGIER          | | [Character]   (amber type badge)   | |
|  | A · ARAGORN  ◀     | |                                    | |
|  | B · BORDEN          | | STORY ARCS                         | |
|  | C · CALDLOW         | | [The Return] [Heir] [+]            | |
|  | D · DAMIEN          | |                                    | |
|  | E · ELARA           | | ALLIES                             | |
|  | E · ETHELRYN (loc)  | | [Eowyn] [Faramir] [+]              | |
|  | ─── EVENTS ───      | |                                    | |
|  | E · ESCAPE FROM ... | | LOCATIONS                          | |
|  | F · FALL OF ELARA   | | [Edoras] [Minas Tirith] [+]        | |
|  | ─── ACTS ───        | |                                    | |
|  | A · ACT I           | | NOTES                              | |
|  | A · ACT II          | | ▼ On the king's name               | |
|  |                     | |   He was always Strider to me      | |
|  | (search bar)        | |   ...                              | |
|  | [filter all types ▾]| | ▼ Crown notes                      | |
|  |                     | |   See [[Eowyn]]'s perspective in   | |
|  +─────────────────────+ |   [[Battle of Pelennor]] for the   | |
|                          |   reluctance theme                 | |
|                          | [+ Add note]                       | |
|                          |                                    | |
|                          | ▼ Details (motivation, body...)    | |
|                          +────────────────────────────────────+ |
|  900 × 600  (sidebar 240 / content flex)                        |
+─────────────────────────────────────────────────────────────────+
```

### Pass 1 — Info Architecture
**Sidebar:** alphabetical entity browser. Notes are NOT in the sidebar (per Lock 2). Type sections separated by faint dividers (`─── EVENTS ───`). Filter dropdown at bottom toggles which entity types appear (default all). Search bar above filter.

**Content area:** name (Fraunces 24px) → type badge → relationship sections (matches existing Connections-First pattern in DESIGN.md line 132) → **Notes section** (new, slots above Details) → collapsible Details (existing). Notes render as collapsible disclosures (▼/▶), one per attached note. Each note shows its body inline; `[+ Add note]` chip at the bottom of the section creates a new note attached via `note_of` relationship.

**Sidebar dim/highlight:** entries whose type is in `currentScope.active*Ids` keep full opacity; others drop to 0.4 opacity (the Wiki sidebar subscriber, Lock 2).

### Pass 2 — Interaction States
| State | Sidebar | Content | Notes section |
|---|---|---|---|
| Loading | Skeleton 8 rows | Spinner | (hidden until entity loads) |
| Empty (no entities) | "Your wiki is empty. Create a Character to begin." + ghost rows | "Pick an entity from the sidebar." | n/a |
| Empty (entity, no notes) | Existing entry highlighted | Entity name + sections render | "No notes yet. [+ Add note]" — single muted dashed-border chip |
| Out-of-scope (Wiki sidebar dim) | Inactive entries 0.4 opacity, hover lifts to 1.0 | Active entry rendered normally | Notes follow parent's scope (Lock 2 union semantics) |
| Error ([[]] resolve fail) | n/a | Body renders with `[[Unknown]]` as literal grey strikethrough text + tooltip "no entity found" | n/a |
| `[[NoteTitle]]` link clicked | Sidebar scrolls + highlights the parent entity | Content navigates to parent | Notes section auto-scrolls to the linked note + highlights briefly (amber pulse 600ms) |

### Pass 3 — User Journey (key flow: hyperlink navigation)
- 5sec: visceral — sidebar lookalike of Apple Mail / Things (familiar). 
- 5min: behavioral — user types `[[Eowyn]]` in Aragorn's body, autocomplete suggests, picks, link saves. 
- 5yr: reflective — wiki becomes the spine; Notes section the catch-all annotation surface; jumping between linked entries feels like Wikipedia.

**[[NoteTitle]] click flow (Lock 15):** content area animates: parent entity loads, Notes section scrolls into view (smooth, 300ms), target note's disclosure auto-expands, 600ms amber pulse on the note's left border. User immediately sees they landed inside the parent, not on a standalone Note page.

### Pass 4 — AI Slop Risk
**Risks:**
- Don't render Notes section as a card grid. It's a series of disclosure rows, text-led.
- Don't put icons next to entity types in the sidebar (the type-prefix `Char ·` text + grouping divider is enough).
- Don't add a hero header on the wiki "welcome screen." Empty state is enough.
- Don't decorate the alphabetical sidebar with a giant first-letter sticky-header treatment unless very subtle (lean: just plain dividers between type groups, no per-letter ALL-CAPS jumbotrons).

### Pass 5 — Design System Alignment
| Element | Token / pattern |
|---|---|
| Sidebar entries | 11px Inter 400, 28px row, type prefix in 9px Inter 600 uppercase tracked |
| Type group divider | 1px `--color-border` + 9px Inter 600 uppercase label |
| Search bar | Reuse the Palette search input pattern (already shipped 2026-05-04) |
| Filter dropdown | 11px Inter 500, native `<select>` styled with `--color-surface-2` bg |
| Entity name | Fraunces 24px (slightly larger than the 20px in CharacterEditor — wiki is the destination, editor is denser) |
| Type badge | 11px Inter 600 uppercase, padding 2px 8px, color `--color-accent` for Character, per-type colors from `--color-type-*` for others |
| Relationship section | Existing Connections-First pattern (DESIGN.md line 132) |
| Notes section heading | 9px Inter 600 uppercase 0.12em tracked, "NOTES" |
| Note disclosure caret | 10px Inter, `--color-text-muted`. Open = ▼, closed = ▶ |
| Note body | Inter 13px, body-mode (line-height 1.55) |
| `[[Entity]]` link rendered | Reuse `.entity-chip` pattern from DESIGN.md line 96 with `--chip-color: var(--color-rel-arc)` (default for entity links) |
| `[[NoteTitle]]` link rendered | Same chip style with `--chip-color: var(--color-rel-note)` (NEW — see Open Decisions) |
| `[[Unknown]]` fallback | Plain text, 13px, `--color-text-muted`, strikethrough, tooltip "no entity found" |
| Out-of-scope dim | `opacity: 0.4` on sidebar row, `transition: opacity 200ms ease` |

### Pass 6 — Responsive + A11y
- Min width 1280px. Window 900×600 default.
- Keyboard: Arrow Up/Down in sidebar; Enter opens. `[`,`]` brackets jump to next/prev letter group. Inside content: Tab cycles relationship chips, Notes disclosures, Details. Cmd/Ctrl+L focuses the body editor.
- Screen reader: sidebar = `<nav>`; type-group dividers as `<h3>`; entity name = `<h1>`; Notes section = `<section aria-label="Notes">`; each note disclosure = `<details><summary>`.
- Touch targets ≥ 44px on disclosure carets and chips.

### Pass 7 — Unresolved
- **`--color-rel-note` token** — what hex? Notes are an *annotation* layer; should be subtle. Lean: warm grey `#a8a29e` (matches `--color-type-note` already in app.css).
- **Sidebar search behavior** — does it filter the sidebar list or does it open Cmd-K instead? Lean: filter sidebar inline (cheaper UX, Cmd-K is a separate global affordance).
- **Type filter dropdown** — multi-select or single-select? Lean: multi-select with checkboxes.
- **Empty state copy** — current DESIGN.md says Wiki empty state is "No notes yet. Create your first note." That's now wrong (Notes aren't navigable). Need rewrite. Lean: "Your wiki is empty. Create a Character or Location to begin." with [+ Open Characters] button.

## Surface 4 — World Map (Phase 1 + 1.5 hex)

### Layout (ASCII) — Phase 1 (regions, no hex)

```
+--[ ● ● ] World Map ────────────────────────────────────────────+
|                                                                 |
|  +────────────────────────────────────────────────────────────+ |
|  |                                                            | |
|  |   (Leaflet canvas: imported bitmap map, pan/zoom)          | |
|  |                                                            | |
|  |        ╱──────╲                                            | |
|  |       │ Edoras │  ◀ active region: glowing amber           | |
|  |        ╲──────╱                                            | |
|  |                                                            | |
|  |              ╱──╲                                          | |
|  |             │Hel│ ◀ inactive: 0.3 opacity                  | |
|  |              ╲──╱                                          | |
|  |                                                            | |
|  +────────────────────────────────────────────────────────────+ |
|                                                                 |
|  Toolbar (top-right floating): [Pan] [Draw region] [Layer ▾]    |
|  Legend (bottom-right): ● Active ○ Inactive (Phase 1.5: hex)    |
|                                                                 |
|  1024 × 720  (full Leaflet, toolbar overlay)                    |
+─────────────────────────────────────────────────────────────────+
```

### Pass 1 — Info Architecture
First: **the map itself** (the whole window, full-bleed). Toolbar is a floating control cluster top-right; never competes with the map. Legend bottom-right, dim until hovered. Active regions glow (amber `--color-accent` ring + 0.13 fill); inactive dim to 0.3 opacity. Region label appears on hover.

**Phase 1.5 hex layer adds:** a toggle in the Layer dropdown ("Show hex grid"). When on, a faint hex SVG overlay renders. Revealed hexes (per `map_hex_reveals.revealed_at`) at full opacity; unrevealed at 0.1.

### Pass 2 — Interaction States
| State | Map | Toolbar | Legend |
|---|---|---|---|
| Loading | Spinner centered | Disabled | Hidden |
| Empty (no map) | "No map yet. [+ Import a bitmap]" centered button | Show only [+] button | Hidden |
| Imported, no regions | Map renders, no overlays. "Draw a region to link to a Location." muted hint top-left | All tools enabled | Hidden |
| Drawing | Cursor crosshair, dashed polygon-in-progress, click to add vertex, Esc cancels | Active tool highlighted | Visible |
| Region linked | Glow on click; popup with Location name + open button | n/a | n/a |
| Error (B2 upload fail) | Inline toast bottom: "Couldn't upload map, try again" + retry | Show only [+] | Hidden |
| Hex layer ON, scrubbing | Revealed hexes solid; unrevealed fading in as scope reveals | n/a | Visible: shows hex meaning |
| Out of scope (active=∅) | All regions dim to 0.3 | n/a | Visible |

### Pass 3 — User Journey (the demo flow)
- 5sec: visceral — open Map app, the user's imported world fills the window, pan/zoom feels native.
- 5min: behavioral — draw a polygon over Edoras, link to the Edoras Location entity. Switch to Timeline, scrub. Edoras glows when active.
- 5yr: reflective — re-reading own story with hex-fog reveal: world unfolds as the playhead moves. The "wow."

**Demo-flow specifically called out in design doc:** open app → import bitmap → draw 3 regions → switch to Timeline → scrub → see regions glow. This MUST work end-to-end with no rough edges.

### Pass 4 — AI Slop Risk
**Risks:**
- Don't put a coloured-card legend with icons. Legend is small, dim, text-led.
- Don't add a "How to use" splash overlay on first import. The hint copy is enough.
- Don't make the toolbar a thick brushed-glass panel. Match the Window.svelte aesthetic — minimal, dark, low-chrome.
- Don't use stock-image map placeholders for empty state. The "[+ Import]" button + 1-line hint is enough.

### Pass 5 — Design System Alignment
| Element | Token / pattern |
|---|---|
| Map background | Leaflet handles bitmap. Canvas bg `--color-desktop` while loading |
| Region polygon (active) | Stroke `--color-accent` 2px, fill `color-mix(in srgb, var(--color-accent) 13%, transparent)` |
| Region polygon (inactive) | Stroke `--color-border` 1px, fill `color-mix(in srgb, var(--color-text-muted) 8%, transparent)`, opacity 0.3 |
| Region label on hover | 12px Inter 500, white text, dark backdrop pill, 4px 8px padding |
| Toolbar | `--color-surface-2` bg, 8px radius, 1px `--color-border`, 4px gap between buttons, 32×32 icon buttons |
| Legend | 10px Inter 400, `--color-text-muted` text, 16×16 swatches |
| Hex layer (Phase 1.5) | SVG, stroke 1px `--color-border`, fill `color-mix(in srgb, var(--color-accent) 4%, transparent)`. Revealed hexes: opacity 1; unrevealed: opacity 0.08 |
| Hex reveal animation | `transition: opacity 800ms ease-out`. New hexes fade in as scope advances |

### Pass 6 — Responsive + A11y
- Min width 1280px (DESIGN.md). Window 1024×720 default; min 800×600.
- Keyboard: arrow keys pan, +/- zoom, R rotate, Esc cancels active draw. Tab cycles toolbar buttons. Enter activates focused region (opens linked Location in EntityDetail).
- Screen reader: map = `<region role="application" aria-label="World map">`; regions are `<button>` elements within an SVG layer with `aria-label="<location name>, <active|inactive>"`.
- **Note:** Leaflet a11y is mediocre by default. Plan to add custom focus-ring overlay on focused regions (visible 2px amber outline) in addition to whatever Leaflet provides.
- Touch targets: regions ≥ 44px hit area when zoomed; toolbar buttons 32×32 visual but 44×44 hit.

### Pass 7 — Unresolved
- **Bitmap import format support** — JPG/PNG/WebP only? SVG too? Lean: JPG/PNG/WebP for v1. SVG = future stretch.
- **Bitmap size limit** — see eng-review failure-mode flag (Issue 12). Lean: 5MB upload cap with clear error, document on the import button hover.
- **Region drawing constraints** — allow self-intersecting polygons? Lean: no; reject at draw-finish with a small toast.
- **Multiple maps per story** — supported? Doc spec'd `world_maps` table (plural). UI: dropdown at top-left to switch active map. Lean: yes, ship multi-map support.

## Surface 5 — Cmd-K Command Palette (Phase 1.5)

### Layout (ASCII)

```
+──────────── (centered overlay, dimmed background) ──────────────+
|                                                                 |
|       +────────────────────────────────────────────────+        |
|       |  > aragor_                                     |        |
|       +────────────────────────────────────────────────+        |
|       |  ▶ Aragorn (Character)                         |        |
|       |    Edoras (Location)                           |        |
|       |    Battle of Helm's Deep (Event)               |        |
|       |    Note: "On the king's name"                  |        |
|       |  ─────────────────────────────────────────     |        |
|       |  Open Settings                                 |        |
|       |  Toggle Theme                                  |        |
|       +────────────────────────────────────────────────+        |
|       |  ↑↓ navigate · ↵ open · esc close              |        |
|       +────────────────────────────────────────────────+        |
|                                                                 |
|       560 × 400 (centered, top 25% offset, dim backdrop)        |
+─────────────────────────────────────────────────────────────────+
```

### Pass 1 — Info Architecture
First: **input field** (the doing). Second: **results** ranked by recency + match strength. Entities first (because navigating to a thing is the most common intent), commands second (separated by 1px divider). Third: **footer hints** (↑↓ ↵ esc) so users learn the shortcuts.

### Pass 2 — Interaction States
| State | Input | Results | Footer |
|---|---|---|---|
| Open (empty input) | Cursor blink | Recent entities (5) + 3 most-used commands | Visible |
| Typing | Live filter as keys press | Fuzzy-ranked, max 12 results | Visible |
| No matches | "..." typed | "No matches. Press Esc to close." muted text | Visible |
| Selected (arrow nav) | Input retains text | Selected row bg `--color-surface-2` + amber 4px left border | Visible |
| Pressed Enter | (briefly highlights) | Selected action fires, palette closes | Hidden during transition |
| Dimmed backdrop | n/a | n/a | Backdrop `rgba(0,0,0,0.5)` |

### Pass 3 — User Journey
- 5sec: visceral — Cmd-K, palette appears centered with focus on input. Familiar (Spotlight, VS Code, Linear).
- 5min: behavioral — typing partial names returns instant matches; user navigates by keyboard only.
- 5yr: reflective — Cmd-K becomes the primary navigation; mouse-clicking sidebar feels slow by comparison.

### Pass 4 — AI Slop Risk
**Risks:**
- Don't add icons next to result rows by default. Type prefix in muted text is enough.
- Don't put a "Powered by ⚡" tagline anywhere.
- Don't render results as cards with shadows. Plain rows.
- Don't add gradients to the input border or backdrop.

### Pass 5 — Design System Alignment
| Element | Token / pattern |
|---|---|
| Backdrop | `rgba(0, 0, 0, 0.5)` + `backdrop-filter: blur(2px)` |
| Palette container | 8px radius, `--color-surface-2` bg, 1px `--color-border`, drop shadow `--window-shadow` |
| Input | Inter 18px 400, 16px padding, transparent bg, `--color-text` color, no visible border (input row's bottom-border serves) |
| Input bottom border | 1px `--color-border` |
| Result row | 11px Inter 400, 32px row height, 12px horizontal padding |
| Type prefix | 9px Inter 600 uppercase, `--color-text-muted`, 8px right margin |
| Selected row | bg `--color-surface-2` (slightly lighter than container), 4px left border `--color-accent`, text `--color-text` |
| Section divider (entities/commands) | 1px `--color-border`, 4px vertical gap |
| Footer | 10px Inter 400, `--color-text-muted`, 8px padding |

### Pass 6 — Responsive + A11y
- Min width 1280px. Palette 560×400 default; min 480×300.
- Keyboard: that's the whole UX. Esc closes. Cmd/Ctrl+K opens (per Lock from hotkey-registry). Arrow up/down navigates. Enter opens. Tab moves focus into the result list (alternative to arrows).
- Screen reader: input `aria-label="Search"`, results = `<ul role="listbox">`, selected has `aria-selected="true"`. Live region announces "N results" as the user types.
- Touch targets: rows ≥ 44px hit area (32 visual + padding).

### Pass 7 — Unresolved
- **Search index strategy** — locked in eng-review Open Q3: in-memory Fuse.js. Confirm token shape: `{ name, type, parentName?, body? }`. Lean: skip body for v1 (just name + type), add body if Cmd-K becomes the primary find).
- **Result limit** — 12 visible? scroll for more? Lean: cap at 12, "show more" affordance only if needed. Most users will refine query.
- **Recent / pinned** — does the palette remember recent picks? Lean: yes, last-5 entities prepended when input is empty.

## Surface 6 — Story Player Mode (Phase 1.5)

### Layout (ASCII) — controls in Timeline

```
+--[ ● ● ] Timeline ────────────────────────────────────────────+
|  Acts ▶  Act I  |  Act II  |  Act III                          |
|  +─────────────────────────────────────────────────────────+   |
|  |                                                         |   |
|  |  (existing timeline rendering)                          |   |
|  |                                                         |   |
|  |        ▼ playhead                                       |   |
|  |        |                                                |   |
|  |  ───────●──────────────────────                         |   |
|  |        ⊃                                                |   |
|  |                                                         |   |
|  +─────────────────────────────────────────────────────────+   |
|                                                                |
|  [⏵ Play]  [⏸] [◀◀] [▶▶]  Speed: [4s/scene ▾]   [×] Exit Player|
|                                                                |
+────────────────────────────────────────────────────────────────+
```

### Pass 1 — Info Architecture
First: the **timeline itself** (unchanged, full visual focus). Player controls dock at the bottom of the Timeline window, below the act tracks. Single row: play/pause/skip + speed dropdown + exit. When Player is OFF, the row collapses (hidden) to preserve current Timeline real estate.

**On entering Player mode** (click ⏵ Play):
- The whole Timeline dims slightly (`opacity: 0.95`) to signal "we're in playback."
- Player dock row appears + slides up.
- Subscriber windows (Map, Wiki sidebar dim) start animating.
- Playhead auto-advances at the user-set speed.

### Pass 2 — Interaction States
| State | Player dock | Playhead | Subscriber windows |
|---|---|---|---|
| Off (default) | Hidden | Manual scrub only | Static (scope-store still drives, just from manual playhead) |
| Player On, paused | Visible, [⏵ Play] highlighted | Stationary | Static (whatever current scope is) |
| Player On, playing | Visible, [⏸] highlighted | Auto-advances at user-set speed | Animate per scope-store updates |
| End of timeline | [⏵ Play] becomes [⤺ Restart] | At end | Static |
| Manual scrub during playback | (keeps playing? lean: pauses) | User-controlled | Update on scrub |
| `prefers-reduced-motion` | Speed clamped to ≥ 8s/scene; subscriber animations crossfade only (no slide/scale) | Same | Subdued animation |

### Pass 3 — User Journey (the demo moment)
- 5sec: visceral — clicks Play, the world lights up: Map regions glow on/off, Wiki sidebar dims/un-dims, Palette shows in-scope characters. The story plays in front of them.
- 5min: behavioral — they pause to read a Note that just became active. Scrub backward to see what they missed. Resume.
- 5yr: reflective — they show this to a friend. Friend gets it instantly. The "wait, what?" reaction the design doc called the demo moment.

This is THE feature that sells the product. Every transition has to feel intentional.

### Pass 4 — AI Slop Risk
**Risks:**
- Don't add a "Now playing: Act II" video-player chyron. The window title + playhead is enough.
- Don't show animated confetti on completion or any other gimmick.
- Don't use a thick gradient background while in Player mode. Just dim the rest.
- Don't put icons in colored circles in the dock. Standard player iconography (▶ ⏸) is enough.

### Pass 5 — Design System Alignment
| Element | Token / pattern |
|---|---|
| Player dock row | 48px tall, `--color-surface-2` bg, 1px `--color-border` top |
| Play/pause button | 32×32 circle, `--color-accent` bg when active, 18px icon (Lucide `play` / `pause`) |
| Skip buttons | 24×24 transparent, 14px icon, hover `--color-surface-2` bg |
| Speed dropdown | 11px Inter 500, `--color-surface` bg, 1px border, 4 options: 2s/4s/8s/16s per scene |
| Exit button | 12px Inter 500, right-aligned, `--color-text-muted` text, hover `--color-text` |
| Player-mode timeline dim | `opacity: 0.95` on existing tracks |
| Subscriber animation | 200ms ease-in-out for opacity changes; 400ms for region glow appearance |
| `prefers-reduced-motion` opacity transitions only, no scale/slide |

### Pass 6 — Responsive + A11y
- Min width 1280px. Player dock fits horizontally; below 720 wide, controls wrap to 2 rows.
- Keyboard: Space toggles play/pause; ←/→ skip back/forward 1 scene; +/- adjust speed; Esc exits player.
- Screen reader: dock = `<div role="toolbar" aria-label="Story Player controls">`; play/pause has `aria-label="Play"` / `"Pause"`; speed dropdown labeled. Live region announces scope changes ("Now in Act II, Scene 3: <active characters>").
- `prefers-reduced-motion`: respected via media query in CSS + a setting in Settings → Appearance to override (always animate / always subdued).

### Pass 7 — Unresolved
- **Where do controls live?** Embedded in Timeline window (above) vs floating overlay vs in Taskbar dock? **Lean: embedded in Timeline,** since Player IS a Timeline mode. Floating overlay clutters; Taskbar is reserved for app launches.
- **Manual scrub during playback** — pause vs co-play? **Lean: pause.** Common video-player convention.
- **Subscriber windows that aren't open** — does Player auto-open Map/Wiki? **Lean: no, only animate already-open windows.** User chooses what to watch.

## Cross-cutting design decisions surfaced

These touch multiple surfaces and need a single decision each:

- **`--color-rel-note`** — token value for the new note-of relationship type. Used by `[[NoteTitle]]` chip color, Notes section accent, and the Notes app folder tree highlight when a folder contains story-attached notes.
- **Out-of-scope dim opacity** — Wiki sidebar, Map regions, Palette filter all use the same dim treatment. **Lean: 0.4 across the board** for visual consistency.
- **Empty-state message tone** — DESIGN.md line 187 sets a warm, actionable tone ("No characters yet. Create one in Characters — it will appear here automatically."). New surfaces (Notes app, Settings, Cmd-K) follow same tone.
- **First-run onboarding** — does opening Settings / Notes / Wiki for the first time show any one-time hint? Lean: no for Settings (self-evident), light text-only hint for Notes ("Brain dump anything"), light text-only hint for Wiki when empty.
