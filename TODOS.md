### 3. Svelte Stores + Cross-Window Navigation

- [x] `src/lib/stores/entities.ts` ‚Äî writable store, load on app init, optimistic update helpers (`createEntity`, `updateEntity`, `deleteEntity`)
- [x] `src/lib/stores/relationships.ts` ‚Äî same pattern; relationships carry semantic `type` values: `"appears_in"`, `"takes_place_at"`, `"caused_by"`, `"allied_with"`, `"rivals"`, `"mentor_of"`, `"located_at"`
- [x] `src/lib/navigation.ts` ‚Äî `openEntity(id: string)` universal navigation function. One window per entity, keyed by `${appId}-${entityId}`. If the window already exists, bring it to front. If not, open a new instance. This means Elara's Character Editor and Damien's Character Editor are independent windows; Arc 1 Timeline and Arc 2 Timeline are independent windows; all can be open simultaneously.
  ```ts
  const ENTITY_APP: Record<EntityType, AppId> = {
    Character: 'character-editor',
    Location:  'world-map',
    Event:     'timeline',
    Act:       'timeline',
    Scene:     'timeline',
    Note:      'wiki',
  };

  export function openEntity(id: string) {
    const entity = get(entities).find(e => e.id === id);
    const appId = ENTITY_APP[entity.type];
    const windowId = `${appId}-${id}`;          // unique per entity
    const existing = get(windows).find(w => w.id === windowId);
    if (existing) {
      windowStore.focus(windowId);               // bring to front, no duplicate
    } else {
      windowStore.open({ id: windowId, appId, entityId: id }); // new window
    }
  }
  ```
- [x] `windowStore` tracks windows as a list of `{ id, appId, entityId, x, y, width, height, minimized, zIndex }`. Windows are keyed by `id` (`${appId}-${entityId}`), not by `appId` ‚Äî multiple instances of the same app type are fully supported.
- [x] Taskbar: group open windows by `appId`. If only one Character Editor is open, show one button. If three are open, show a grouped button with a count badge ‚Äî clicking it opens a mini-picker listing Elara, Damien, Scout. Prevents taskbar clutter with many open entity windows.

## Phase 2 ‚Äî Desktop Shell

### 4. Window Manager

- [x] `src/lib/components/WindowManager.svelte` ‚Äî renders all open windows, manages z-index stack
- [x] `src/lib/components/Window.svelte` ‚Äî draggable, resizable, minimizable; props: `title`, `appId`, `initialPosition`
- [x] **Window spawn position:** Cascading offset. Track a module-level `spawnOffset` counter in `windowStore`. Each new window opens at `{ top: 60 + (spawnOffset % 8) * 24, left: 60 + (spawnOffset % 8) * 24 }`, then increment counter. Resets modulo 8 so it doesn't push off a 1280px viewport. Default window size: 320√ó480px.
- [x] `src/lib/stores/windows.ts` ‚Äî open/close/focus/minimize state for all windows
- [x] `src/lib/components/Taskbar.svelte` ‚Äî app buttons, minimized window list, story position slot (placeholder)
- [x] `src/lib/components/Desktop.svelte` ‚Äî background, icon dock
- [x] **Minimum viewport:** Below 1280px wide, replace the entire app with a centered lockout message: "betwixt-and-between works best on a larger screen. Expand your browser window to continue." Implemented as a CSS media query wrapper + a `<TooSmall>` component ‚Äî no JS required.
- [x] **Keyboard shortcuts** (handle in `WindowManager.svelte` via `keydown` on `document`):
  - `Ctrl+W` ‚Äî close the currently focused window
  - `Ctrl+Tab` / `Ctrl+Shift+Tab` ‚Äî cycle forward/backward through open windows (update `zIndex` stack)
  - `Escape` ‚Äî dismiss any open inline form (relationship label prompt, etc.)
  - `Tab` / `Shift+Tab` ‚Äî navigate between EntityLink chips within a section
  - `Enter` ‚Äî activate the focused chip (calls `openEntity(id)`)
  - `Ctrl+K` ‚Äî reserved for future command palette (no-op for now, don't let browser default handle it)
- [x] **Focus rings:** All interactive elements (chips, buttons, window controls) must have a visible focus ring using `outline: 2px solid var(--color-accent)` ‚Äî never `outline: none` without a replacement.
- [x] **First-open / empty state:** Clean dark desktop with app icons in the dock. No windows open on fresh launch. A one-time tutorial overlay identifies each dock icon with a label + one-line description (e.g. "Characters ‚Äî manage your cast", "Story Graph ‚Äî see how they connect"). Overlay is dismissible and never shows again after first close. The empty desktop itself shows a subtle centered line: "Your story starts here." (disappears once the first entity is created). This preserves the OS aesthetic while orienting new users.
- [x] Playwright tests:
  - Open window ‚Üí appears on screen
  - Click taskbar button ‚Üí window comes to front (z-index check)
  - Drag window ‚Üí position updates
  - Minimize ‚Üí window hidden, taskbar button still present

- [x] On window open: fetch entities + canvas_positions from API, build tldraw snapshot, call `editor.store.loadSnapshot(snapshot)`
- [x] On node move (tldraw interaction): write new x/y to `/api/canvas-positions`
- [x] On edge create (tldraw interaction): animate edge into place, then immediately show an inline relationship mini-form overlaid on the edge midpoint ‚Äî type dropdown (allies/rivals/mentor/family/caused_by/located_at/appears_in) + optional label text field + Save/Cancel. On Save: write relationship to `/api/relationships` with type + label, update edge label in tldraw. On Cancel: delete the edge. This is the core 'aha moment' interaction ‚Äî the connection becomes semantically meaningful in the same gesture it's drawn.

- [x] On entity delete: remove shape from tldraw on next render (handled by store subscription)
- [x] Pin tldraw version in `package.json` ‚Äî do not upgrade without re-testing `loadSnapshot()`

## Phase 4 ‚Äî Feature Windows

### 6. Character Editor

Every window is a wiki page. Connections are primary, fields are secondary.

- [x] `src/lib/components/apps/CharacterEditor.svelte` ‚Äî opens focused on a `Character` entity (receives `focusEntityId` from window store)
- [x] **Layout (connections-first):** Name + entity type badge at top. Then relationship groups: STORY ARCS, ALLIES, RIVALS, LOCATIONS, KEY EVENTS ‚Äî each as a labeled section with `EntityLink` chips and a `[+]` button to add a new relationship of that type. Fields (motivation, arc summary, notes) are collapsed below a "‚ñº Details" disclosure ‚Äî expandable on click. This makes the character's web of connections the primary view, not a form.
- [x] Save fields ‚Üí PATCH `/api/entities/[id]` ‚Üí optimistic store update
- [x] **Linked entities panel** (primary, above fold):
  - Story arcs / Acts / Scenes this character appears in ‚Üí click any ‚Üí `openEntity(arcId)` ‚Üí Timeline opens at that arc
