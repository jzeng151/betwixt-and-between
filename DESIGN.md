# betwixt-and-between — Design System

Generated by /plan-design-review on 2026-04-22. All decisions here were made explicitly — nothing is a default.

---

## Palette — Midnight Ink

A candlelit writer's room at night. Near-black surfaces, warm amber accents, off-white text. Nothing competes with the content.

```css
:root {
  /* Surfaces */
  --color-desktop:    #0d0f14;  /* desktop background */
  --color-surface:    #161920;  /* window background */
  --color-surface-2:  #1c1f28;  /* title bar, raised elements */
  --color-border:     #2a2d35;  /* separators, window borders */

  /* Text */
  --color-text:       #e8e0d0;  /* primary — warm off-white */
  --color-text-muted: #6b7280;  /* secondary — labels, subtitles */

  /* Accent */
  --color-accent:     #c8942a;  /* amber gold — active states, clock, badges */

  /* Relationship type colors — used on EntityLink chips */
  --color-rel-arc:    #c8942a;  /* story arcs */
  --color-rel-ally:   #2dd4bf;  /* allies */
  --color-rel-rival:  #ef4444;  /* rivals */
  --color-rel-mentor: #818cf8;  /* mentor */
  --color-rel-family: #86efac;  /* family */
  --color-rel-loc:    #818cf8;  /* locations */
  --color-rel-event:  #94a3b8;  /* events */
  --color-rel-other:  #94a3b8;  /* default */
}
```

## Typography

Two typefaces. Fraunces carries the narrative weight. Inter handles everything functional.

| Role | Font | Weight | Size | Color |
|------|------|--------|------|-------|
| Entity names (character, location, arc, event) | Fraunces | 400 | 20px | `--color-text` |
| Window title | Inter | 500 | 11px | `--color-text-muted` — uppercase, 0.06em tracked |
| Section labels | Inter | 600 | 9px | `--color-text-muted` — uppercase, 0.12em tracked |
| Body / chip text | Inter | 400 | 11px | contextual |
| Buttons / UI labels | Inter | 500 | 12px | contextual |

**Load:**
```html
<link href="https://fonts.googleapis.com/css2?family=Inter:wght@400;500;600;700&family=Fraunces:ital,opsz,wght@0,9..144,300;0,9..144,400;1,9..144,300&display=swap" rel="stylesheet">
```

**Rules:**
- Entity names always use Fraunces. Everything else uses Inter.
- Never use `system-ui` or `-apple-system` as the primary font — pick Inter explicitly.
- Body text minimum 11px. Never below.
- Never `outline: none` without a visible replacement focus ring.

## EntityLink Chips

Clickable references to other entities. Every chip calls `openEntity(id)` on click. Color is determined by relationship type.

```css
.entity-chip {
  display: inline-flex;
  align-items: center;
  padding: 3px 10px;
  border-radius: 20px;
  font-family: var(--font-ui);
  font-size: 11px;
  font-weight: 400;
  background: color-mix(in srgb, var(--chip-color) 13%, transparent);
  color: var(--chip-color);
  border: 1px solid color-mix(in srgb, var(--chip-color) 33%, transparent);
  cursor: pointer;
  transition: background 0.15s ease;
}

.entity-chip:hover {
  background: color-mix(in srgb, var(--chip-color) 22%, transparent);
}

.entity-chip:focus-visible {
  outline: 2px solid var(--color-accent);
  outline-offset: 2px;
}

/* Add chip (the [+] button in each section) */
.entity-chip-add {
  background: transparent;
  color: var(--color-text-muted);
  border: 1px dashed var(--color-border);
}
```

Set `--chip-color` to the appropriate `--color-rel-*` variable based on relationship type.

## Character Editor Layout — Connections-First

Entity name and relationships are primary. Fields (motivation, notes) are secondary, collapsed by default.

```
+--[ ● ● ] Character ─────────────────+
|                                      |
|  Elara Voss          (Fraunces 20px) |
|  [Character]         (amber badge)   |
|                                      |
|  STORY ARCS                          |
|  [The Fall of Elara] [Redemption] [+]|
|                                      |
|  ALLIES                              |
|  [Scout] [Mira] [+]                  |
|                                      |
|  RIVALS                              |
|  [Damien] [+]                        |
|                                      |
|  LOCATIONS                           |
|  [Ashenveil] [The Keep] [+]          |
|                                      |
|  ──────────────────────────────────  |
|  ▼ Details (motivation, notes)       |
+──────────────────────────────────────+
```

The same connections-first layout applies to all entity types — Location, Event, Act/Scene, Note. Each leads with its name (Fraunces), its type badge, then relationship groups, then collapsible fields.

## Story Graph — Edge Draw Interaction

When a user draws an edge between two nodes in tldraw, immediately overlay an inline relationship form at the edge midpoint:

```
  [Elara] ────────── [Damien]
               ↓
  +--[ What is this? ]----------+
  |  Type: [rivals        ▾]   |
  |  Label: [optional text]    |
  |  [Save]  [Cancel]          |
  +------------------------------+
```

- Type dropdown options: `allies`, `rivals`, `mentor`, `family`, `appears_in`, `caused_by`, `located_at`, `other`
- Label is optional free text
- Save → writes to `/api/relationships`, updates edge label in tldraw
- Cancel → removes the edge
- Escape key → same as Cancel

## Empty States

Every app window has a warm, actionable empty state. "No items found." is never acceptable.

| Window | Empty state message | Primary action |
|--------|--------------------|--------------:|
| Story Graph | "No characters yet. Create one in Characters — it will appear here automatically." + ghost dimmed example nodes | → opens Characters app |
| Timeline | "No events yet. Add an Act or Event to begin your story." | [+ Add Act] button |
| Wiki | "No notes yet. Create your first note." | [+ New Note] button |
| World Map | "No locations yet. Add a location to place your story." | [+ Add Location] button |
| Desktop (first run) | "Your story starts here." — disappears after first entity is created | dock tutorial overlay |

## First-Run Experience

On fresh install (no entities in DB), the desktop shows:

1. Clean dark desktop, dock icons visible at bottom
2. One-time tutorial overlay: each dock icon gets a label + one-line description ("Characters — manage your cast", "Story Graph — see how they connect", etc.)
3. Centered on the canvas: "Your story starts here." in `--color-text-muted`
4. Tutorial overlay is dismissible; never shows again after first close
5. The "Your story starts here." line disappears once the first entity is created

## Keyboard Shortcuts

Handled globally in `WindowManager.svelte` via `keydown` on `document`.

| Shortcut | Action |
|----------|--------|
| `Ctrl+W` | Close the currently focused window |
| `Ctrl+Tab` | Cycle forward through open windows |
| `Ctrl+Shift+Tab` | Cycle backward through open windows |
| `Escape` | Dismiss any open inline form or modal |
| `Tab` / `Shift+Tab` | Navigate between EntityLink chips within a section |
| `Enter` | Activate the focused chip (calls `openEntity(id)`) |
| `Ctrl+K` | Reserved — command palette (future, no-op for now) |

All interactive elements must have a visible focus ring: `outline: 2px solid var(--color-accent); outline-offset: 2px`.

## Viewport

Minimum supported width: **1280px**.

Below 1280px, replace the entire app with:

```
+----------------------------------+
|                                  |
|   betwixt-and-between            |
|                                  |
|   Works best on a larger         |
|   screen. Expand your browser    |
|   window to continue.            |
|                                  |
+----------------------------------+
```

Implemented as a CSS `@media (max-width: 1279px)` wrapper showing a `<TooSmall>` component. No broken layout, no cramped windows.

## Interaction States

### Loading
- Window content area: dark background + centered spinner (amber, 24px)
- Skeleton shimmer on text placeholders where content is expected

### Error / Save Fail
- Inline, non-blocking: small error message below the relevant action button
- Text: `--color-rel-rival` (#ef4444), 11px
- Never a full-screen error — always recoverable in place

### Saving (optimistic)
- No loading indicator — stores update instantly via optimistic update
- If the API call fails: revert the store and show inline error

## What This Design Is Not

- Not a generic SaaS dashboard
- Not card-grid layouts
- Not light mode (dark-only in v1)
- Not mobile (desktop browser only)
- Not decorative blobs or gradient backgrounds
- Not system-ui as the primary font
